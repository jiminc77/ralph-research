Summary · 한 줄 요약

정답은 "과제마다 다르다".
그리고 진짜 주인공은 "조합"이다.

로프를 곧게 펼 때는 어디를 잡느냐가, 끝점을 옮길 때는 어느 방향으로 미느냐가 더 중요했습니다. 하지만 두 과제 모두에서 가장 큰 효과는 위치와 방향의 궁합(상호작용)이었습니다. 단, 저자들 스스로 밝히듯 이 수치들은 품질 검증을 완전히 통과하지 못한 파일럿(예비) 증거입니다.

위치 우세
곧게 펴기는 '위치' 싸움

ΔWH = +0.041. 좋은 지점을 잡기만 하면 어느 방향으로 당겨도 대체로 통합니다. 히트맵이 가로 줄무늬인 이유죠.

방향 우세
끝점 옮기기는 '방향' 싸움

ΔWH = −0.130. 어디를 잡든 미는 방향이 목표를 향해야 합니다. 히트맵이 세로 줄무늬로 뒤집힙니다.

판정 유보
한 번 접기는 무승부

ΔWH = −0.034, 신뢰구간이 0을 포함해 balanced/inconclusive. 대신 위치·방향의 궁합이 결과를 지배합니다.

중요한 각주 — 이건 "확정 결론"이 아닙니다

사전에 등록한 데이터 품질 기준(가드)을 어떤 실행 조건도 통과하지 못했기 때문에, 논문은 모든 수치를 서술적 파일럿 증거로 스스로 강등했습니다. 자세한 이야기는 정직성 리포트에서.

Background · 왜 어려운가

로프를 다루는 로봇의 두 가지 고민

로프·케이블·전선 같은 변형 가능한 선형 물체(DLO)는 로봇에게 여전히 난제입니다. 모양이 무한히 변하고(고차원 상태), 만져야만 움직일 수 있고(접촉 매개), 한 점을 잡아도 나머지는 마음대로 안 됩니다(저작동).

그래서 실용 시스템은 하이브리드 액션을 씁니다 — ① 어디를 잡을지(접촉 위치 p) 고르고, ② 어떻게 움직일지(모션 명령 u) 정하는 2단 구조입니다. 문제는 기존 벤치마크가 둘을 합친 최종 점수만 매긴다는 것. 성공해도 그것이 위치를 잘 골라서인지, 방향을 잘 골라서인지 알 수 없습니다.

이 귀속(attribution)이 중요한 이유: 다음 모델링 노력을 더 촘촘한 접촉 지도에 쓸지, 더 풍부한 모션 생성에 쓸지, 둘의 공동 추론에 쓸지가 여기서 갈리기 때문입니다.

좁은 주장 경계 (Narrow claim)
두 인자는 표현력이 대칭이 아닙니다. 위치 인자는 로프 전체 길이를 8등분해 커버하지만, 방향 인자는 고정된 이동 거리의 평면 8방향뿐(크기·수직·회전 없음). 그래서 이 논문의 비교는 "어디 vs 어떻게" 일반론이 아니라 "접촉 위치 선택 vs 고정 크기 평면 방향 선택"이라는 좁은 질문입니다 — 저자들이 직접 강조하는 경계입니다.
Method · 실험 방법

학습된 정책을 조사하는 대신,
모든 경우의 수를 직접 실행했다

이 논문은 새 알고리즘을 제안하지 않습니다. 대신 고전 통계의 완전요인설계(모든 조합을 빠짐없이 실험)를 액션 공간에 적용했습니다. 모델도, 추측도 없이 "직접 개입"으로 측정합니다.

1
상태 표준화

로프를 정해진 초기 모양으로 놓고 속도를 0으로 리셋, 완전히 정지할 때까지 안정화. 배치 오차는 10⁻⁴ m 이내여야 합니다(이 가드가 나중에 사건의 발단이 됩니다).

2
64조합 전부 실행

잡기 → 이동 → 놓기 → 안정화 프리미티브로 8위치 × 8방향을 모두 실행. GPU 병렬 로드 시뮬레이터에서 초당 5.42개의 유효 셀을 처리합니다.

3
진전도 Y 기록

목표 모양과의 오차가 얼마나 줄었는지를 정규화: Y = 1이면 한 번에 해결, 0이면 제자리, 음수면 오히려 악화.

4
표에서 네 값 읽기

완성된 8×8 표는 "무엇을 아는 게 얼마나 값진가"를 그대로 알려줍니다. 아래 수식과 데모에서 확인하세요.

# 한 걸음 진전도 — 목표 오차가 얼마나 줄었나 (비율)
Y(p, u) = (D_before − D_after(p, u)) / D_before

# 8×8 표에서 읽는 네 가지 "지식 수준"
v0  = mean(Y)             # 아무 칸이나 찍기 (운)
vP  = max_p mean_u Y      # 잡을 위치만 잘 알 때 (최고의 행)
vU  = max_u mean_p Y      # 밀 방향만 잘 알 때 (최고의 열)
vPU = max_(p,u) Y         # 둘 다 알 때 (최고의 칸)

# 핵심 지표
Δ_WH = E[vP − vU]            # 양수 → 위치가 더 값짐 · 음수 → 방향이 더 값짐
I    = E[vPU − vP − vU + v0] # 상호작용 — 어느 한쪽만으론 설명 안 되는 "짝"의 가치
직접 눌러보기 — 네 가지 지식 수준

개념 예시 데이터입니다(t1b 유형의 얼룩덜룩한 표). 버튼을 누르면 각 전략이 표에서 무엇을 "아는" 것인지 표시됩니다.

잡는 위치 p (8행)
미는 방향 u (8열, 45° 간격)

실험 조건(후보 실행 체계): 이동 거리와 들어올림 높이는 C0(0.10 m·낮음)–C3까지 4가지로 고정해 두고 우선순위 규칙으로 선택하도록 사전등록 — 효과 크기를 보고 고르는 것을 원천 차단했습니다.

Results · 결과

인자의 중요도는 과제를 따라 뒤집힌다

Interaction · 상호작용

그런데 진짜 큰 효과는 '짝'이었다

곧게 펴기의 상호작용은 I = +0.228, 한 번 접기는 +0.154 — 각각 어느 단독 인자보다 큽니다. 표 분산에서 상호작용이 차지하는 몫이 86%, 76%에 달하죠. 최고의 칸은 "좋은 위치 + 좋은 방향"의 합이 아니라 특정한 짝이라는 뜻입니다. 끝점 옮기기만 예외로 상호작용이 거의 없습니다(+0.007).

둘 다 알 때의 이득(vPU − v0)은 세 과제 모두 +0.30 ~ +0.40 — 아무렇게나 행동할 때보다 목표 오차를 30~40%p 더 줄입니다.

해석 주의 2가지
① "방향 우세" ≠ "방향 지식이 더 유익". 잘못된 방향은 적극적으로 해를 끼치고(Y < 0) 잘못된 위치는 그저 무익한(Y ≈ 0) 경우가 많아, 방향 우세는 피해 회피를 반영할 수 있습니다. 실제로 표 전체 평균 v0가 세 과제 모두 음수(−0.151 / −0.142 / −0.066)입니다. ② 라벨은 실행 체계에 한정. 이동 거리·들어올림 조건(C0–C3)을 바꾸면 ΔWH의 부호가 일관되지 않아, 위 라벨들은 C0 조건에서의 관찰일 뿐입니다.
Integrity · 정직성 리포트

이 논문의 백미:
실패를 1페이지부터 공개한다

저자들은 실험 전에 규칙을 공증(사전등록)해 두었습니다 — "무효 테이블 비율이 2%를 넘는 실행 조건은 선택하지 않는다." 그런데 실제 무효율은 21.9% (42/192). 원인은 가설이 아니라 리셋 시스템의 결함이었습니다: 6개 상태에서 로프 재배치 오차가 기준(10⁻⁴ m)의 최대 64배까지 벗어난 것. 규칙은 규칙이므로, 어떤 조건도 공식 선택되지 않았습니다.

Guard
가드 — 사전에 정한 문지방

64칸 중 하나라도 잡기 실패·발산·재배치 오차가 나면 표 전체를 무효 처리. 사후에 기준을 완화하거나 셀만 골라내는 것을 금지했습니다.

Quarantine
격리 — 오염 데이터 차단

무효 테이블 42개는 평균에 절대 섞지 않고 격리·공개. 분석에 쓰인 표들 자체는 신호 품질 검사(반복 재현성 1.000, 안정화 수렴 99.6%)를 통과했습니다.

Downgrade
강등 — 결론의 지위 낮추기

후보 선택이 효과 크기를 보지 않는(effect-blind) 규칙이었기에, "확증 결과"가 아닌 서술적 파일럿 증거로의 강등이 신뢰할 수 있게 성립합니다.

그 밖의 사전등록 이탈도 모두 명시: 상태 수 축소(등록 96/64 → 실제 12/13/15), 히든 분할 미실행, 주장 범위 축소 등. 부풀리는 대신 깎아서 보고하는 드문 사례입니다.

Lessons · 시사점

그래서 무엇을 바꿔야 하나

액션 설계자에게
  • 개수가 같다고 공평한 게 아니다. 8위치는 로프 전체를, 8방향은 모션 공간의 얇은 조각만 커버 — 귀속 연구는 이런 표현력 감사를 함께 공개해야 합니다(16×16·크기 추가 후속 실험 등록됨).
  • 인자 중요도는 과제의 속성이다. "어디(where)"에 용량을 몰아주는 아키텍처(밀집 affordance 맵 + 얇은 모션 헤드)는 과제 의존적 편향을 안고 시작합니다.
  • 상호작용이 크면 따로 점수 매기지 마라. 3과제 중 2개에서 짝의 가치가 지배적 — 접촉·모션의 공동 스코어링을 유지할 근거입니다.
벤치마크 설계자에게
  • 리셋 충실도는 1급 속성. 테이블별 배치 검사가 없었다면 이 결함은 모든 추정치를 조용히 오염시켰을 것 — 시뮬레이터는 결정적 재배치를 노출하고 검사해야 합니다.
  • 최댓값 통계엔 노이즈 플로어 대조가 필요. 표에서 뽑은 최댓값은 위로 치우칩니다. 제로 변위 대조(|Y| 중앙값 1.4×10⁻³)가 바닥선 역할을 했습니다.
  • 가드 임계값은 규모를 내다봐야. 2% 기준은 이 파일럿 규모에선 후보당 무효 표 1개 미만만 허용 — 국소 결함 하나로 전멸합니다. 표본 크기에 비례하는 임계나 격리 의미론이 필요합니다.
한계 (논문이 직접 나열)

과제당 12~15개 상태의 개발 분할뿐인 파일럿 · 시뮬레이터 1종, 로프 파라미터 1종, 잡기 현실성 비활성화 · 한 걸음(greedy) 지표라 다단계 정책 결론 불가 · 방향 인자에 크기·수직·회전 없음 · 실행 조건 간 부호 불안정 · 넓은 구간은 검정력 부족이지 "동등하다"는 증거가 아님.

Glossary · 용어 사전

10개 용어만 알면 원문도 읽힌다

Deformable Linear Object (DLO)변형 가능한 선형 물체

로프·케이블·전선처럼 길고 흐물흐물한 물체. 상태가 고차원이고 접촉해야만 움직여서 로봇에겐 고난도.

Quasi-static준정적

휙 던지는 동역학 없이, 움직였다가 완전히 멈춘 상태끼리 비교하는 설정. 인과를 깔끔하게 따지기 위한 선택.

Canonicalization정준화

매 실험 전에 로프를 정확히 같은 초기 상태로 되돌리는 절차. 이 재배치의 오차가 이번 연구 최대의 복병이었음.

Full Factorial Design완전요인설계

인자들의 모든 조합(여기선 8×8=64)을 빠짐없이 실행하는 고전 실험설계. 각 인자의 몫과 상호작용을 분해할 수 있음.

Preregistration사전등록

실험 전에 가설·규칙·기준을 못박아 공개하는 것. "결과 보고 규칙 바꾸기"를 차단하는 신뢰 장치.

Validity Guard · Invalid Table가드 · 무효 테이블

데이터 품질 문지방. 한 칸이라도 실패하면 표 전체를 버리는 엄격 규칙 — 42/192개가 여기 걸려 격리됨.

ΔWH (Where − How)위치−방향 대비

"위치만 알 때 가치(vP) − 방향만 알 때 가치(vU)"의 상태 평균. 양수면 위치 우세, 음수면 방향 우세.

Interaction (I)상호작용

둘 다 알 때의 가치에서 각 단독 지식의 몫을 뺀 나머지 = "짝"의 가치. 이 논문의 숨은 주인공.

Candidate Execution Regime후보 실행 체계

이동 거리·들어올림 높이 등 나머지 자유도를 통째로 고정한 설정(C0~C3). 효과를 안 보고 우선순위로만 선택.

Cluster Bootstrap클러스터 부트스트랩

상태 단위로 재표집해 불확실성을 재는 방법. 과제·초기모양으로 층화해 10⁴번 재표집, 동시 95% 구간 산출.
